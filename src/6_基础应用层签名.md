# 模块六：基础应用层——ReSolveD+ 与环签名

## 一、概述

本模块是将底层 $\Pi^t_{dD-Rep}$ 协议应用到具体密码学原语的第一层。包含两种签名方案：

1. ReSolveD+：基于 RSD（Regular Syndrome Decoding） 困难问题的数字签名方案。公钥是一个随机矩阵 $B$ 和其对应的综合症 $y$；私钥是一个随机秘密 $x$，满足 $y = B \cdot \text{RE}(x)$。
2. 环签名（Ring Signature）：签名者可以从一个公钥集合（环）中匿名地签署消息，使验证者确信签名者属于该集合，但无法确定具体是哪一个。这里利用 Merkle 树累加器 + AFS 双矩阵哈希构造环。

## 二、ReSolveD+ 签名

### 2.1 数学基础：RSD 问题

RSD（Regular Syndrome Decoding） 是编码理论中的一个 NP-hard 问题：给定矩阵 $B \in \mathbb{F}\_2^{n \times m}$ 和综合症 $y \in \mathbb{F}\_2^n$，寻找一个正则词 $x$（其汉明重量极低，结构上由 $c$ 比特块映射为独热向量）使得：

$$
y = B \cdot \text{RE}(x) \quad (\text{在 } \mathbb{F}\_2 \text{ 上})
$$

其中 $\text{RE}$ 是正则编码函数（见模块二），它将 $k$ 比特的输入 $x$ 映射为 $m = \frac{k}{c} \cdot 2^c$ 比特的独热拼接。

### 2.2 密钥结构

```rust
pub struct PublicKey {
    pub b_cols: StreamingMatrixCols,   // B 矩阵（流式列生成）
    pub y: MainBitVec,                 // 综合症 y = B · RE(x)
    pub k: usize,                      // 秘密 x 的比特长度
    pub c: usize,                      // 正则编码块大小
}

pub struct SecretKey {
    pub x: MainBitVec,                 // 秘密 x（witness）
}
```

密钥生成：
1. 随机选择种子 $\sigma$ → 生成 $B = \text{PRG}(\sigma) \in \mathbb{F}\_2^{n \times m}$
2. 随机选择 $x \in \mathbb{F}\_2^k$
3. 计算 $y = B \cdot \text{RE}(x)$
4. 公钥 = $(B, y, k, c)$，私钥 = $x$

### 2.3 ZK 证明语句

证明的目标是向 Verifier 零知识地证明"我知道 $x \in \mathbb{F}\_2^k$ 使得 $y = B \cdot \text{RE}(x)$"。

等价地，Prover 需要证明 $x$ 满足以下约束系统 $g_i(x) = 0$：

$$
g_i(x) = y_i + \sum_{j=1}^m b_{i,j} \cdot f_j(x) = 0, \quad \forall i \in \{1, \dots, n\}
$$

其中 $f_j(x)$ 是 $x$ 的正则编码 $\text{RE}(x)$ 的第 $j$ 个输出位，可以表示为 degree-$c$ 的 ANF 多项式（参见模块四）。

### 2.4 约束构建：`build_resolved_plus_constraints`

对于每个约束方程 $g_i(x) = 0$，代码使用三步流程构建 ANF 约束系统：

第一步：获取每个正则编码索引的本地单项式（`local_monomials`）

对于正则编码输出索引 $\text{idx}$（0 到 $2^c-1$），其 ANF 展开包含一组单项式（来自 `re_table[idx]` 中为 1 的列）。

第二步：反向映射 + BitVec 批量 XOR

```rust
// table_idx -> 包含它的 re_idx 列表
let mut table_idx_to_re_idx = vec![Vec::new(); domain_size];
for re_idx in 0..domain_size {
    for &table_idx in &local_monomials[re_idx] {
        table_idx_to_re_idx[table_idx].push(re_idx);
    }
}

// 对每个 (block_idx, table_idx)，计算 B·RE 中该单项式的系数行
for block_idx in 0..block_count {
    let block_offset = block_idx * domain_size;
    for table_idx in 0..domain_size {
        let mut presence = empty_col.clone();
        for &re_idx in &table_idx_to_re_idx[table_idx] {
            presence ^= &b_cols[block_offset + re_idx];  // BitVec XOR

        global_monomial_presence.push(presence);
    }
}
```

这里的关键洞察：ANF 单项式 $X_{j_1} X_{j_2} \cdots X_{j_d}$ 在约束 $g_i$ 中的系数由 $B_{i, \cdot}$ 与 ANF 系数表的组合决定。通过位向量批量 XOR，避免了逐行逐列的 $\mathcal{O}(n \cdot m \cdot 2^c)$ 暴力循环。

第三步：构建最终方程

```rust

    }
}
// 处理常数项 y
for (row_idx, bit) in y.iter().by_vals().enumerate() {

```

最终输出的 ANF 约束系统：`Vec<Vec<Monomial>>`，共 $n$ 个多项式（对应 $y$ 的 $n$ 个比特），每个多项式的度数为 $c$，变量范围为 $k$。

### 2.5 签名流程（`sign_with_rng`）

```
Algorithm: ReSolveD+ Sign(pk, sk, msg)
Input:  pk = (B, y, k, c), sk = x, msg
Output: Signature σ

1.  Statement Digest:
     absorb(pk.k, pk.c, B.seed, y, msg) → statement_digest

2.  VOLE Commit:
     VoleitHProver::commit(ReSolveD128Var2, statement_digest, k, c)
     → (u, v, BAVC_state)

3.  Correction:
     corr = x ⊕ u → CommitmentMessage

4.  Challenges:
     absorb(..., corr) → relation_seed → χ[0..n]

5.  χ' 聚合:
     chi_prime = compute_chi_prime_streaming_f128(χ, B)
     const_term = Σ_{i: y_i=1} χ_i

6.  Implicit Evaluation + Response:
     ProverRef::respond_resolved_plus_implicit(χ, chi_prime, const_term, c, n)
     → ProofMessage { blinded_coeffs[0..c] }

7.  FAEST Opening:
     VoleitHProver::finalize(proof) → VoleitHTranscript

8.  Return σ = (CommitmentMessage, ProofMessage, VoleitHTranscript)
```

### 2.6 验证流程（`verify`）

```
Algorithm: ReSolveD+ Verify(pk, msg, σ)

1.  VOLE Reconstruct:
     VoleitHVerifier::reconstruct(..., σ.commitment, σ.proof, σ.vole)
     → (Δ, q)

2.  Absorb Correction:
     corrected_q = q + correction · Δ

3.  Recompute Challenges:
     reconstruct relation_seed → χ[0..n]

4.  χ' 聚合 (同 Prover):
     chi_prime = compute_chi_prime_streaming_f128(χ, B)
     const_term = Σ_{i: y_i=1} χ_i

5.  Verify Core Equation:
     implicit_evaluate(chi_prime, const_term, corrected_q, c, Δ) + Π q_aux
     ?= evaluate_univariate(proof.blinded_coeffs, Δ)
```

## 三、环签名（Ring Signature）

### 3.1 数学基础：Merkle 累加器的 ZK 证明

环签名的核心思路是使用 Merkle 哈希树 累加环中所有公钥。签名者需要在零知识下证明自己知道一条从叶子（自己的密钥材料）到根（公开的 Merkle 根 $u_0$）的认证路径。

AFS 双矩阵哈希 用于 Merkle 树的每一层。对于层 $\theta$，给定左子节点 $v_\theta$ 和右子节点 $w_\theta$，双亲节点由下式算出：

$$
v_{\theta-1} = h_B(v_\theta, w_\theta) = B_0 \cdot \text{RE}(v_\theta) \oplus B_1 \cdot \text{RE}(w_\theta)
$$

其中 $B_0, B_1 \in \mathbb{F}\_2^{n \times m}$ 是两个独立的伪随机矩阵。

关键点：在每一层，签名者还需要证明路径的方向——即当前节点是左子还是右子。这通过一个方向比特 $j_\theta \in \{0,1\}$ 编码在约束中。

### 3.2 密钥结构

```rust
pub struct RingPublicKey {
    pub b0_cols: StreamingMatrixCols,   // B_0 矩阵
    pub b1_cols: StreamingMatrixCols,   // B_1 矩阵
    pub root_u: MainBitVec,             // Merkle 根 u_0
    pub ell: usize,                     // Merkle 树高度（路径长度）
    pub n: usize,                       // 节点大小（比特）
    pub c: usize,                       // 正则编码块大小
}

pub struct RingSecretKey {
    pub witness: MainBitVec,            // 完整 witness 拼接
}
```

### 3.3 Witness 结构

Witness 包含 $\ell$ 层 Merkle 路径的所有秘密信息。总长度为：

```rust
fn expected_witness_len(pk: &RingPublicKey) -> usize {
    pk.ell + 2 * pk.ell * pk.n + 2 * pk.n
}
```

其具体拼接结构为：

```
witness = [j_0..j_{ell-1}]        // ell 个方向比特
        + [v_1] + [w_1]           // 第 1 层的左右兄弟节点 (2n 比特)
        + [v_2] + [w_2]           // 第 2 层的左右兄弟节点 (2n 比特)
        + ...
        + [v_ell] + [w_ell]       // 第 ell 层 (2n 比特)
        + [x_0] + [x_1]           // 两个叶子秘密 (2n 比特)
```

```
              u_0 (Merkle Root, 公开)
             /   \
           /       \
        v_1         w_1
       /   \       /   \
     v_2   w_2   ...   ...
     / \   / \
   ... ... ... ...
    /             \
  x_0             x_1  (叶子秘密)
```

其中 $j_\theta$ 表示从第 $\theta-1$ 层到第 $\theta$ 层的路径方向。例如 $j_1 = 0$ 表示 $u_0 = h_B(v_1, w_1)$（方向正常），$j_1 = 1$ 表示 $u_0 = h_B(w_1, v_1)$（左右交换）。

### 3.4 约束构建：`build_ring_signature_constraints`

约束系统包含 $(\ell + 1) \cdot n$ 个多项式（$\ell$ 层每层 $n$ 个 + 叶子层 $n$ 个）。

#### 3.4.1 每层的约束方程

对于第 $\theta$ 层（$0 \le \theta < \ell$），我们知道：

$$
u_{\theta-1} = \begin{cases}
h_B(v_\theta, w_\theta) & \text{若 } j_\theta = 0 \\
h_B(w_\theta, v_\theta) & \text{若 } j_\theta = 1
\end{cases}
$$

用 ANF 表示，这等价于：

$$
u_{\theta-1} = h_B(v_\theta, w_\theta) \cdot (1 + j_\theta) \; \oplus \; h_B(w_\theta, v_\theta) \cdot j_\theta
$$

代码中，`build_pair_hash_terms` 为两种顺序分别计算 ANF 单项式集合 $A_i$（当 $j=0$）和 $B_i$（当 $j=1$）。然后通过 `symmetric_difference` 提取差异，用方向比特 $j_\theta$ "提升"差异单项式的度数：

```rust
let a_terms = build_pair_hash_terms(b0_cols, b1_cols, v_offset, w_offset, ...);
let b_terms = build_pair_hash_terms(b0_cols, b1_cols, w_offset, v_offset, ...);

for row_idx in 0..n {
    let mut equation = a_terms[row_idx].clone();
    // 对称差中的单项式需要乘以方向变量 j_theta（提升一度）
    for monomial in a_terms[row_idx].symmetric_difference(&b_terms[row_idx]) {
        let mut lifted = monomial.clone();
        lifted.push(direction_idx);   // 加入 j_theta 变量
        xor_insert(&mut equation, normalize_monomial(lifted));
    }
    // 处理常数项或父节点引用
    if theta_idx == 0 {
        // 顶层：目标根是公开的 root_u
        if root_u[row_idx] { xor_insert(&mut equation, Vec::new()); }
    } else {
        // 非顶层：目标父节点是上一层的 v 值
        xor_insert(&mut equation, vec![v_offset(theta_idx-1) + row_idx]);
    }
}
```

这精确实现了上述 $u_{\theta-1}$ 的 ANF 约束。

#### 3.4.2 叶子层约束

叶子层方程为：

$$
v_{\ell-1} = h_B(x_0, x_1) = B_0 \cdot \text{RE}(x_0) \oplus B_1 \cdot \text{RE}(x_1)
$$

```rust
let leaf_terms = build_pair_hash_terms(b0_cols, b1_cols, x0_offset, x1_offset, ...);
for (row_idx, mut equation) in leaf_terms.into_iter().enumerate() {
    // 目标父节点是第 ell-1 层的 v 值
    xor_insert(&mut equation, vec![v_offset(ell-1) + row_idx]);
    constraints.push(equation.into_iter().collect());
}
```

### 3.5 隐式求值引擎（针对环签名的配对蝴蝶）

由于环签名需要同时处理两个矩阵 $B_0$ 和 $B_1$，代码构建了专用的 配对 χ' 数据结构：

```rust
pub(crate) struct RingImplicitData<F> {
    layer_pair_hashes: Vec<PairHashChi<F>>,  // 每层的 χ' 对
    leaf_pair_hash: PairHashChi<F>,           // 叶子层的 χ' 对
    root_const_term: F,                       // 根节点常数项
}
```

`build_ring_implicit_data` 并行计算每层的 χ' 值：

```rust
fn compute_pair_hash_chi(challenges, b0_cols, b1_cols) -> PairHashChi<F128b> {
    let (left, right) = compute_chi_prime_pair_streaming_f128(challenges, b0_cols, b1_cols);
    PairHashChi { left, right }
}
```

这里的 `compute_chi_prime_pair_streaming_f128` 是模块四中配对 χ' 聚合的实现——它同时计算 $B_0$ 和 $B_1$ 的 χ' 向量，并在可能的情况下使用配对蝴蝶（`xor_pair_32!`）同时处理左右矩阵。

核心求值函数 `evaluate_ring_implicit_from_affine` 实现：

```
Algorithm: evaluate_ring_implicit_from_affine(challenges, data, affine, point, ell, n, c)
Input:  affine (仿射基), point (p)
Output: Σ_i χ_i · g̃_i(p)

1. total = 0
2. for theta in 0..ell:
     // 左右子节点的仿射基
     left_affine  = affine[v_offset(theta) .. v_offset(theta)+n]
     right_affine = affine[w_offset(theta) .. w_offset(theta)+n]
     
     // 配对蝴蝶求值：同时计算 direct 和 swapped 两个分量
     (eval_a, eval_b) = evaluate_pair_hash_swapped_from_affine(
         pair_hash[theta], left_affine, right_affine, c, point)
     
     // 加入路径方向 j_theta 的贡献
     j_theta = affine[theta]  // 第 theta 个方向比特
     total += eval_a · (point + j_theta) + eval_b · j_theta
     
     // 加入父节点约束（根恒定项 / 中间节点）
     if theta == 0:
         total += root_const_term · point^{c+1}
     else:
         total += Σ challenges · affine[parent_v] · point^{c}
     
3. // 叶子层
   leaf_eval = evaluate_pair_hash_direct_from_affine(
       leaf_pair_hash, x0_affine, x1_affine, c, point)
   total += leaf_eval · point
   total += Σ challenges · affine[leaf_v] · point^{c}
   
4. return total
```

注意这里约束的度数比 ReSolveD+ 高 1（$c+1$ 而非 $c$），因为方向比特 $j_\theta$ 额外贡献了一度。

### 3.6 环签名完整签名流程

```
Algorithm: Ring Sign(pk, sk, msg)
Input:  pk = (B0, B1, root_u, ell, n, c), sk = witness, msg
Output: RingSignature σ

1. Statement Digest:
     absorb(pk.ell, pk.n, pk.c, B0.seed, B1.seed, root_u, msg) → statement_digest

2. VOLE Commit:
     witness_len = ell + 2·ell·n + 2·n
     VoleitHProver::commit(FAEST128F, statement_digest, witness_len, c+1)
     → (u, v, BAVC_state)

3. Correction:
     corr = witness ⊕ u → CommitmentMessage

4. Challenges:
     absorb(..., corr) → relation_seed → χ[0 .. ell·n + n]

5. Build RingImplicitData:
     pair_hash_chi[θ] = compute_chi_prime_pair_streaming_f128(χ[θ·n .. (θ+1)·n], B0, B1)

6. Advanced Implicit Response:
     ProverRef::respond_advanced_implicit(c+1, expected_constraints, χ,
         |affine, point| evaluate_ring_implicit_from_affine(pk, χ, data, affine, point))
     → ProofMessage { blinded_coeffs[0..c+1] }

7. FAEST Opening → VoleitHTranscript

8. Return σ
```

### 3.7 Witness 正确性验证

测试 `test_ring_signature_rejects_witness_mutations_in_implicit_path` 验证了三种违反情况的检测：

1. 叶子秘密篡改（修改 $x_0$）：破坏 $v_{\ell-1} = h_B(x_0, x_1)$
2. 路径选择比特篡改（修改 $j_\theta$）：破坏路径方向
3. 内部节点篡改（修改 $v_\theta$）：破坏 $u_{\theta-1} = h_B(v_\theta, w_\theta)$

三种情况均被验证拒绝，证明了约束系统的完备覆盖。

## 四、总结对比

| 特征 | ReSolveD+ | 环签名 (RS) |
|------|-----------|------------|
| 底层困难问题 | RSD | 2-RNSD + Merkle 累加器 |

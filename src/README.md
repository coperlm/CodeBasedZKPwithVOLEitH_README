# Code-Based Zero-Knowledge from VOLE-in-the-Head: 核心代码与架构解析报告

## 报告导读

本报告从数学关系、证明协议和应用语句三个层面解释实现。代码采用 Rust 编写，FAEST 的 VOLE/BAVC 部分通过固定的 C 参考实现接入。全文以源码、测试和仓库中的论文复现实验说明为依据；未在这些材料中确认的安全结论不在此处外推。

### 模块一：底层数学与矩阵运算

本模块是整个 ZKP 系统的数学基石与性能瓶颈突破口。

- $GF(2^{128})$ 运算：通过 `swanky_field` 的 `F128b` 表示有限域元素，硬件路径可使用 CLMUL/PMULL。
- 列流式矩阵：对由种子确定的矩阵按列生成，避免默认物化整个矩阵。

👉 [点击查看：1_底层数学与矩阵运算.md](./1_底层数学与矩阵运算.md)

### 模块二：核心密码学原语

提供两种基于编码的密码学原语，它们是构建上层 ZK 签名系统的代数骨架。

- AFS 哈希：以 regular encoding 和二元矩阵乘法为核心的编码型哈希关系，并提供流式/掩码求值路径。
- McEliece 关系：为 GroupSig 的加密一致性提供约束所需的编码、矩阵和密文操作。

👉 [点击查看：2_核心密码学原语.md](./2_核心密码学原语.md)

### 模块三：VOLEitH 状态机与 FFI

实现了 VOLE-in-the-Head 范式所需的功能，并负责跨语言的安全集成。

- 跨语言接口：接入 FAEST 的 VOLE commit/open/reconstruct 和 BAVC/GGM 状态。
- Rust 封装：负责长度、返回状态、资源释放和结构布局检查；这些检查不等同于独立安全证明。

👉 [点击查看：3_VOLEitH状态机与FFI.md](./3_VOLEitH状态机与FFI.md)

### 模块四：多项式约束生成

系统的理论创新核心，将正则编码等操作编译为布尔多项式，并进行极速求值。

- 非线性正则编码代数化：利用布尔立方体上的子集关系，将 regular encoding 写成 degree-$c$ 的 ANF 约束。
- 蝴蝶隐式求值：利用求和交换和多线性折叠，避免显式展开每个 regular-encoding 输出。

👉 [点击查看：4_多项式约束生成.md](./4_多项式约束生成.md)

### 模块五：ZK 核心证明协议

实现 $\Pi^t_{dD-Rep}$ 核心协议，负责具体的零知识证明与验证逻辑。

- 线性因子乘积盲化：将 packed auxiliary 的线性因子乘积加入响应多项式。
- Fiat-Shamir 变换：以带标签的 Keccak-256 transcript 派生关系挑战和 opening challenge；其安全性仍需结合协议模型分析。

👉 [点击查看：5_ZK核心证明协议.md](./5_ZK核心证明协议.md)

### 模块六：基础应用层签名

将底层协议转化为具体的密码学签名方案，包含两种基础签名。

- ReSolveD+ 标准签名：基于 RSD 困难问题，证明正则编码约束。
- 环签名 (RS)：通过证明双矩阵 AFS 哈希链方程，完成 Merkle 树累加器的成员资格匿名认证。

👉 [点击查看：6_基础应用层签名.md](./6_基础应用层签名.md)

### 模块七：高级应用层签名

在环签名基础上，引入加密与电路约束，实现具有复杂访问控制的顶级签名方案。

- 群签名 (GS)：将 McEliece 相关密文一致性写入 ANF 约束系统。
- 属性基签名 (FDABS)：将布尔电路中的门关系和属性关系写入约束系统；具体的生命周期管理不属于在线证明路径。

👉 [点击查看：7_高级应用层签名.md](./7_高级应用层签名.md)

### 模块八：参数优化与基准测试

系统在参数选择、序列化体积及并发执行上的工程优化。

- 安全评估：显式计算组合挑战项和 Delta 挑战项，记录 grinding 对工作因子的影响，但不把它直接当作 soundness bit 数。
- 并发与序列化：只作为实现层优化，具体速度和签名大小以实验记录为准。

# 模块十：性能测试结果

本章记录当前版本在 i5-13500H 上的端到端性能。计时覆盖签名生成及编码，以及签名解码和验证；每个测试重复 10 次，去掉最高和最低各一次后取平均值。

## 测试环境

| 项目 | 配置 |
|------|------|
| CPU | Intel Core i5-13500H（12 核 16 线程） |
| 编译 | Rust 1.97.0，`--release`，`target-cpu=native` |
| 并行设置 | `RAYON_NUM_THREADS=16`，`RUST_TEST_THREADS=1` |
| 加速指令 | AES-NI、PCLMULQDQ |
| 源码版本 | `8dbd1c8`；FAEST v2.0.5 |

## 端到端结果

| 方案 | 参数 | 签名并编码 (ms) | 解码并验证 (ms) | 签名体积 |
|------|------|----------------:|----------------:|----------:|
| ReSolveD+ | `n=453,c=3` | 15.727 | 9.715 | 3.08 KiB |
| RingSig | `ell=5` | 20.033 | 14.704 | 34.90 KiB |
| RingSig | `ell=10` | 34.196 | 22.867 | 59.92 KiB |
| Scope-LRS（扩展） | `ell=5` | 21.355 | 15.188 | 35.06 KiB |
| Scope-LRS（扩展） | `ell=10` | 33.946 | 24.538 | 60.08 KiB |
| Batch Scope-LRS（扩展，10 条） | `ell=5` | 168.146 | 111.611 | 306.64 KiB |
| GroupSig | `ell=5,c_e=6` | 20.878 | 17.212 | 50.40 KiB |
| GroupSig | `ell=10,c_e=6` | 34.435 | 25.455 | 75.38 KiB |
| FDABS | `ell=5,L=128,K=512` | 21.450 | 13.540 | 34.15 KiB |
| FDABS | `ell=10,L=128,K=512` | 33.508 | 21.809 | 59.17 KiB |

Batch Scope-LRS 的表项为一次生成或验证 10 条签名的总时间，对应每条签名平均为 16.815 ms 和 11.161 ms，平均体积为 30.66 KiB。

## 结果解读

- RingSig、GroupSig、FDABS 在多核下完成一次签名约需 20--34 ms，验证约需 14--25 ms。
- 从 `ell=5` 增加到 `ell=10` 时，认证路径和约束规模增加，运行时间与签名体积相应上升。
- ReSolveD+ 的关系规模较小，线程调度开销较为显著；其余较大关系可利用并行计算缩短端到端时间。

模块七：高级应用层——群签名与全动态属性基签名 (FDABS)

一、概述

本模块构建在环签名（模块六）的基础上，增加了两个重要的密码学功能：

  1. 群签名（Group Signature, GS）：除了环签名的匿名性外，群签名允许一个群管理员在必要时撤销匿名性，追踪出签名者的身份。这通过 McEliece 加密将身份信息 $j$ 和消息 $m$ 嵌入密文 $(c_1, c_2)$ 实现。

  2. 全动态属性基签名（Fully Dynamic Attribute-Based Signature, FDABS）：签名者拥有一个属性向量，只有当属性满足某个布尔电路策略（Policy） 时才能生成有效签名。代码将布尔电路编码为 AND 门约束，并利用 Merkle 树累加器证明属性集的有效性。

二、群签名（Group Signature）

2.1 安全问题域

群签名同时证明三个断言：

断言安全目标实现机制
签名者属于群匿名性Merkle 树累加器（同环签名）
签名者的身份是 $j$可追踪性McEliece 加密 $c_1 = G_1 \cdot \binom{r_1}{\text{bin}(j)} \oplus \text{RE}(e’_1)$
消息绑定不可伪造性$c_2 = G_2 \cdot \binom{r_2}{\text{bin}(m)} \oplus \text{RE}(e’_2)$

2.2 Witness 结构

群签名的 witness 在环签名 witness 基础上增加了加密随机数和噪声种子：

witness = [j_0..j_{ell-1}]                    // ell 个方向比特
        + [v_1] + [w_1]                       // Merkle 路径节点
        + ...
        + [v_ell] + [w_ell]
        + [x_0] + [x_1]                       // 叶子秘密
        + [r_1]                                // McEliece 随机数 1 (ke-ell 比特)
        + [r_2]                                // McEliece 随机数 2 (ke-ell 比特)
        + [e'_1]                               // 噪声种子 1 (k 比特)
        + [e'_2]                               // 噪声种子 2 (k 比特)

总长度为：

#![allow(unused)]
fn main() {
fn expected_witness_len(pk: &GroupPublicKey) -> Result<usize, String> {
    Ok(pk.ell + 2 * pk.ell * pk.n + 2 * pk.n   // Merkle 部分 (同环签名)
       + 2 * (pk.ke - pk.ell)                    // 随机数 r1, r2
       + 2 * pk.k)                               // 噪声种子 e'_1, e'_2
}
}

2.3 加密约束的构建

群签名约束系统 = 环签名约束 + 两个 McEliece 加密约束。

#![allow(unused)]
fn main() {
// 1. 复用环签名的 Merkle 路径约束
let mut constraints = build_ring_signature_constraints(
    b0_cols, b1_cols, root_u, ell, n, c_hash, hash_re_table
)?;

// 2. 添加 McEliece 加密约束 c1
for row_idx in 0..ne {
    let mut equation = BTreeSet::new();
    // 线性部分: G1 · (r1 || bin(j))
    append_linear_matrix_row_terms(
        &mut equation, g1_cols, 0..r_len, offsets.r1_start, row_idx);
    append_linear_matrix_row_terms(
        &mut equation, g1_cols, r_len..r_len+ell, offsets.j_start, row_idx);
    // 噪声部分: RE(e'_1)
    append_re_terms(&mut equation, row_idx, offsets.e1_prime_start, ...);
    // 常数项: c1_bits
    if c1_bits[row_idx] { xor_insert(&mut equation, Vec::new()); }
    constraints.push(equation.into_iter().collect());
}

// 3. 相同的结构用于 c2
}

加密约束的 ANF 形式：

对于密文 $c_1$ 的第 $i$ 个比特，约束为：

$$ c_{1,i} + \sum_{j: G_{1,i,j}=1} (r_1 | \text{bin}(j))_j + \text{RE}(e’_1)_i = 0 $$

其中：

  • append_linear_matrix_row_terms 处理 线性部分 $G \cdot \binom{r}{\text{bin}(j)}$：遍历 $G$ 矩阵的第 $i$ 行、涉及的列范围，每一列为 1 的位置对应一个线性变量
  • append_re_terms 处理正则噪声 $\text{RE}(e’)$：$\text{RE}(e’)_i$ 在约束层面被展开为多个 degree-$c_e$ 的 ANF 单项式的 XOR 求和（通过 append_re_terms 注入每个单项式）
  • 最后翻转常数项使方程等于 0

这与公式 $c = G \cdot \binom{u}{m} \oplus \text{RE}(e’)$ 完美对应——只不过这个关系被编码在约束方程 $g_i(\cdot) = 0$ 中，而不是直接计算。

2.4 签名流程

Algorithm: Group Sign(pk, sk, msg)
Input:  pk = (B0, B1, G1, G2, root_u, ...), sk = membership_witness, msg
Output: GroupSignature

1.  生成一次性随机数 r1, r2, e'_1, e'_2
2.  将 sk.membership_witness + (r1, r2, e'_1, e'_2) 拼接为完整 witness

3.  Statement Digest:
     absorb(pk.ell, pk.n, pk.ke, pk.ne, pk.k, pk.te,
            pk.c_hash, pk.c_e, B0.seed, B1.seed, G1.seed, G2.seed, root_u, msg)

4.  加密并行计算:
     g_input1 = (r1 || bin(j)), g_input2 = (r2 || bin(j))
     linear1 = linear_matrix_vector_mul_streaming_ct(G1, g_input1)
     linear2 = linear_matrix_vector_mul_streaming_ct(G2, g_input2)
     re_e1 = regular_encode_ct(e'_1, c_e)
     re_e2 = regular_encode_ct(e'_2, c_e)
     c1 = linear1 ⊕ re_e1
     c2 = linear2 ⊕ re_e2

5.  VOLE Commit → Correction → Challenges

6.  Implicit Data:
     Merkle χ'（同环签名）
     G1 χ' = compute_chi_prime_streaming_f128(χ[c1_range], G1)
     G2 χ' = compute_chi_prime_streaming_f128(χ[c2_range], G2)

7.  Advanced Implicit Response:
     自定义求值器 evaluate_group_implicit_from_affine
     → 合并 Merkle 环 + 加密约束的隐式求值

8.  FAEST Opening → VoleitHTranscript

9.  Return σ = (CommitmentMessage, ProofMessage, VoleitHTranscript, c1_bits, c2_bits)

注意：密文 $(c_1, c_2)$ 被显式包含在签名中（Verifier 需要它们来验证约束）。由于这些密文本身就是公开值（任何知道公钥的人都可以验证其结构），这不影响匿名性。

三、全动态属性基签名 (FDABS)

3.1 安全问题域

FDABS 同时证明三个断言：

断言实现机制
属性向量 $a$ 已被群管理员认证Merkle 叶哈希 $v_\ell = B_\text{leaf} \cdot \text{RE}(a | \text{rand})$
叶哈希的汉明重量为奇数$\text{wt}(v_\ell) \equiv 1 \pmod{2}$（奇偶性约束）
属性 $a$ 满足布尔电路策略AND 门约束和最终输出约束
属性属于已认证集合Merkle 累加器路径（同环签名）

3.2 密钥结构

#![allow(unused)]
fn main() {
pub struct FdabsPublicKey {
    pub b0_cols: StreamingMatrixCols,     // Merkle 哈希矩阵 B0
    pub b1_cols: StreamingMatrixCols,     // Merkle 哈希矩阵 B1
    pub b_leaf_cols: StreamingMatrixCols, // 叶子哈希矩阵 B_leaf
    pub root_u: MainBitVec,               // Merkle 根
    pub circuit: BooleanCircuit,          // 策略电路
    pub ell: usize,                       // Merkle 深度
    pub n: usize,                         // 节点大小
    pub attr_len: usize,                  // 属性长度
    pub randomness_len: usize,            // 随机化长度
    pub c: usize,                         // 正则编码块大小
}

pub struct BooleanCircuit {
    pub num_input_wires: usize,      // 输入线数 = attr_len
    pub num_internal_wires: usize,   // AND 门数量
    pub and_gates: Vec<AndGate>,     // [left, right, output]
    pub output_wire: usize,          // 最终输出线
}

pub struct AndGate {
    pub left_wire: usize,    // 左输入线索引
    pub right_wire: usize,   // 右输入线索引
    pub output_wire: usize,  // 输出线索引
}
}

3.3 Witness 结构

witness = [j_0..j_{ell-1}]                    // ell 个方向比特
        + [v_1] + [w_1]                       // Merkle 路径节点
        + ...
        + [v_ell] + [w_ell]
        + [a_0..a_{attr_len-1}]                // 属性向量
        + [rand_0..rand_{randomness_len-1}]    // 随机化比特
        + [gate_output_0..gate_output_{G-1}]   // 每个 AND 门的输出比特

总长度：

#![allow(unused)]
fn main() {
fn expected_witness_len(pk: &FdabsPublicKey) -> usize {
    pk.ell + 2 * pk.ell * pk.n    // Merkle 路径
        + pk.attr_len              // 属性
        + pk.randomness_len        // 随机化
        + pk.circuit.num_internal_wires  // AND 门输出
}
}

3.4 与环签名的关键区别：单矩阵叶哈希

FDABS 中，叶节点由单个矩阵 $B_\text{leaf}$ 计算，而非环签名的双矩阵配对：

$$ v_\ell = B_\text{leaf} \cdot \text{RE}(a | \text{rand}) $$

这是因为 FDABS 没有 “左右兄弟节点” 的交换概念——叶节点是属性 + 随机数的直接哈希。

#![allow(unused)]
fn main() {
// fdabs.rs 中: build_hash_terms（单矩阵版本）
let leaf_terms = build_hash_terms(
    b_leaf_cols,                    // 只有一个矩阵
    offsets.attr_start,             // 输入起始 = 属性位置
    attr_len + randomness_len,      // 输入长度 = 属性 + 随机数
    c, &local_monomials, n,
    "leaf hash matrix",
)?;
}

3.5 叶奇偶性约束

FDABS 要求叶哈希的汉明重量为奇数。这是一个重要的合法性标记——它防止攻击者使用一个无效的（但 ANF 上满足的）叶节点替代真正的认证叶节点。

实现为单个方程：

#![allow(unused)]
fn main() {
let mut leaf_parity = BTreeSet::new();
for row_idx in 0..n {
    xor_insert(&mut leaf_parity, vec![leaf_v_offset + row_idx]);
}
xor_insert(&mut leaf_parity, Vec::new());  // + 1（常数项）
constraints.push(leaf_parity.into_iter().collect());
}

数学上表示：

$$ \left(\bigoplus_{i=0}^{n-1} v_{\ell,i}\right) + 1 = 0 \quad\Longleftrightarrow\quad \text{wt}(v_\ell) \equiv 1 \pmod{2} $$

测试 test_fdabs_constraints_reject_even_leaf_weight 验证了当叶哈希的汉明重量为偶数时约束被违反。

3.6 NAND 门二次约束（名为 AndGate，实际上实现 NAND）

代码中的门使用结构体名 AndGate，但约束实际实现的是 NAND（与非）门。对于每个门，要求 $x_{\text{out}} = \neg(x_{\text{in1}} \land x_{\text{in2}})$，在 $\mathbb{F}_2$ 上的 ANF 约束为：

$$ x_{\text{out}} + x_{\text{in1}} \cdot x_{\text{in2}} + 1 = 0 $$

验证：

  • $x_{\text{in1}} = 1, x_{\text{in2}} = 1$ 时：$x_{\text{out}} = 0$ ✓ (NAND)
  • $x_{\text{in1}} = 1, x_{\text{in2}} = 0$ 时：$x_{\text{out}} = 1$ ✓ (NAND)
  • $x_{\text{in1}} = 0, x_{\text{in2}} = 1$ 时：$x_{\text{out}} = 1$ ✓ (NAND)
  • $x_{\text{in1}} = 0, x_{\text{in2}} = 0$ 时：$x_{\text{out}} = 1$ ✓ (NAND)

（注意：NAND 门构成的电路与 AND 门一样具有表达能力——任何布尔电路都可以用 NAND 门单独构建。AndGate 结构体名反映了论文中的记法，但约束的实际代数实现是 NAND。）

#![allow(unused)]
fn main() {
// fdabs.rs 中 AND 门约束
for gate in &circuit.and_gates {
    let left_idx = wire_to_witness_index(offsets, circuit, gate.left_wire);
    let right_idx = wire_to_witness_index(offsets, circuit, gate.right_wire);
    let output_idx = wire_to_witness_index(offsets, circuit, gate.output_wire);
    constraints.push(vec![
        normalize_monomial(vec![left_idx, right_idx]),  // x_in1 · x_in2
        vec![output_idx],                                // + x_out
        Vec::new(),                                      // + 1 (常数)
    ]);
}
}

这对应于多项式的三个单项式：$X_{\text{in1}}X_{\text{in2}} + X_{\text{out}} + 1 = 0$。这是NAND 门（与非门）的 ANF 约束：当 $X_{\text{in1}} = 1, X_{\text{in2}} = 1$ 时，方程要求 $X_{\text{out}} = 0$；其他情况要求 $X_{\text{out}} = 1$。

3.7 最终输出约束

电路的最后要求是输出线 $x_{\text{out}}$ 必须为 1（策略满足）：

#![allow(unused)]
fn main() {
let output_idx = wire_to_witness_index(offsets, circuit, circuit.output_wire);
constraints.push(vec![vec![output_idx], Vec::new()]);
}

即 $X_{\text{output}} + 1 = 0$，等价于 $X_{\text{output}} = 1$。

3.8 完整的约束系统清单

FDABS 约束总计 $(\ell+1) \cdot n + G + 2$ 个方程：

约束类型数量来源
Merkle 层路径$\ell \cdot n$build_pair_hash_terms（同环签名）
叶子哈希约束$n$build_hash_terms（单矩阵）
叶奇偶性1$(\bigoplus v_\ell) + 1 = 0$
NAND 门（代码中名为 AndGate）$G$$x_{\text{out}} + x_{\text{in1}}x_{\text{in2}} + 1 = 0$
最终输出1$x_{\text{output}} + 1 = 0$

3.9 隐式求值引擎（FDABS 版本）

evaluate_fdabs_implicit_from_affine 函数在 respond_advanced_implicit 中作为自定义求值器使用。其核心结构为：

Algorithm: evaluate_fdabs_implicit_from_affine(pk, challenges, data, affine, point)
Input: affine (仿射基), point (p)
Output: Σ_i χ_i · g̃_i(p) (所有约束在 p 点的 χ-加权和)

1. Merkle 环部分 (同环签名):
   total += Σ_θ [eval_a·(p+j_θ) + eval_b·j_θ + 父节点约束]

2. 叶子哈希约束:
   total += implicit_evaluate_extended_from_affine_no_const(
       leaf_hash_chi,
       affine[attr..attr+attr_len+randomness_len],
       c, p, p)        // 额外乘子 = p (因此处是单矩阵, 度数对齐)
   total += 父节点约束

3. 叶奇偶性约束:
   total += χ_parity · (Σ affine[leaf_v] · p^{c} + p^{c+1})

4. NAND 门约束（对应代码中 `AndGate`）:
   for each gate:
     total += χ_gate · (affine[left]·affine[right] · p^{c-1}
                        + affine[output] · p^{c}
                        + p^{c+1})

5. 最终输出约束:
   total += χ_output · (affine[output] · p^{c} + p^{c+1})

6. return total

3.10 签名流程

Algorithm: FDABS Sign(pk, sk, msg, info, epoch)
Input:  pk = (B0, B1, B_leaf, root_u, circuit, ...), sk = witness, msg, info, epoch
Output: FdabsSignature

1. Statement Digest:
     absorb(pk.ell, n, attr_len, randomness_len, c,
            circuit wires + gates,
            B0.seed, B1.seed, B_leaf.seed, root_u,
            msg, info, epoch)

2. VOLE Commit → Correction → Challenges

3. Implicit Data:
     Merkle χ' (配对哈希, 每层计算 left/right χ')
     B_leaf χ' (单矩阵 χ')

4. Advanced Implicit Response:
     自定义求值器 evaluate_fdabs_implicit_from_affine
     → 合并 Merkle + 叶子 + 奇偶性 + NAND 门 + 最终输出的隐式求值

5. FAEST Opening → VoleitHTranscript

6. Return σ

信息字段 info 和 epoch 使 FDABS 支持动态撤销和策略更新——签名还绑定到特定时期 epoch 和上下文 info。
